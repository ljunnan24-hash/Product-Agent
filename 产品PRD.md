# Product Agent 产品 PRD

版本：MVP v1  
日期：2026-06-21  
目标：用最低开发和发布成本验证“普通创作者愿意把产品材料交给 AI 做产品诊断，并认可诊断价值”。

## 1. 产品一句话

Product Agent 是一个产品诊断 Agent。

用户只需要上传产品 README、产品介绍 PDF、PRD、页面截图或调研材料，再补充一句自然语言说明。Agent 会自动读取材料、提取链接、抓取网页证据，判断产品定位、用户任务、价值主张、体验、信任、转化、发布表达和产品潜力，并给出下一步验证行动。

## 2. 核心假设

AI 产品创作者缺的不是更多表单和模板，而是一个能读懂材料、主动拆解问题、给出高信噪比反馈的产品合伙人。

需要验证的信号：

1. 用户是否愿意上传自己的 README、产品介绍 PDF 或页面材料。
2. 用户是否认为报告“说中了产品内部问题”，而不是只评价好不好看。
3. 用户是否愿意把诊断结果发到社交媒体求反馈。
4. 用户是否认可“这个产品有没有潜力”的判断。
5. 用户是否愿意为更深度的人工增强诊断付费。

## 3. 目标用户

优先用户：

- 正在做 AI 产品的独立开发者和 one-person company 创始人。
- 有 demo、产品介绍或落地页，但不确定定位和表达是否成立的人。
- 准备发 X、即刻、Product Hunt、Reddit、Hacker News、小红书的产品创作者。
- 想用社交媒体低成本验证产品方向的人。

核心痛点：

- 产品能跑，但一句话说不清到底帮谁解决什么。
- README 或产品介绍讲了很多功能，但用户看完不知道为什么现在要用。
- 页面、文案、截图和 CTA 没有形成同一个产品主张。
- 缺少可信证据、使用场景、替代方案比较和转化路径。
- 不知道下一步应该先改产品、改文案、补证据，还是先发出去测试。

## 4. 产品原则

- 一个对话入口。
- 一个上传材料处。
- 不要求用户选择类型、填写复杂表单或配置流程。
- Agent 自动读材料、拆问题、找参考、排优先级。
- 展示的是 workflow trace 和 tool use，不展示模型隐藏思维链。
- 报告必须具体、可执行、能帮助用户获得下一轮信号。

## 5. 用户流程

1. 用户进入首页。
2. 用户上传 README、产品介绍 PDF、PRD、截图或调研材料。
3. 用户可选输入一句说明，例如“帮我判断这个产品有没有潜力”。
4. Agent 自动读取材料、抓取 README 中的公开链接，并在配置搜索 API 时搜索外部网页。
5. 用户进入报告页，看到产品潜力、市场证据、产品诊断、参考对象、下一步行动和执行轨迹。
6. 用户下载分享卡片或申请更深入的人工产品诊断。

## 6. Agent 判断框架

高手判断产品时，不是先看页面是否漂亮，而是先看产品是否成立。

Product Agent 的判断顺序：

1. Product thesis：产品承诺了什么、给谁、在什么情境下发生。
2. Customer job：用户想完成什么进步，今天用什么替代方案。
3. Problem intensity：问题是否高频、痛、急、值钱或强身份相关。
4. Product risks：价值风险、可用性风险、可行性风险、商业可行性风险。
5. Clarity and trust：用户是否能快速理解、相信并知道下一步。
6. Market and distribution fit：目标用户是否可触达，发布话术是否匹配渠道。
7. Craft and focus：材料是否聚焦核心价值，是否删掉了噪音。
8. Next action priority：下一步做什么能最快获得市场学习。

其中“Market and distribution fit”和“产品潜力分”不再依赖模型主观判断，而由 Evidence Agent 单独负责调研。详见：

- `EvidenceAgent证据调研系统PRD.md`

Evidence Agent 的核心职责：

- 把产品命题拆成关键假设。
- 为每个假设生成查询矩阵。
- 大量搜索支持证据和反证。
- 抽取 Evidence Cards。
- 计算 evidence confidence。
- 把最大证据缺口转化成下一步验证实验。

产品判断不是一次性报告，而是一个闭环：

> 定命题 -> 查证据 -> 校准置信度 -> 给决策 -> 设实验 -> 回收实验结果 -> 更新判断

因此报告页不是终点。报告页必须明确告诉用户：

- 当前哪些判断有证据支撑。
- 哪些只是模型推断或待验证假设。
- 为什么现在不能更高置信。
- 下一步做哪个最小实验能最快提高置信度。
- 实验完成后，如何把结果回填并更新判断。

## 7. MVP 输入材料

P0 支持：

- README / Markdown / TXT。
- 产品介绍 PDF。
- PRD 或 pitch deck PDF。
- Landing page / App 页面截图。
- 品牌视觉、海报或社媒图。

P0 限制：

- 单次最多 6 份材料。
- 单个文件 12MB 以内。
- PDF 读取前 8 页、最多约 12000 字。
- 图片使用本地 canvas 提取尺寸、亮度、对比度、饱和度、主色和边缘密度；DeepSeek 当前按文本模型接入。
- README 中出现的公开 URL 会被抓取；未配置搜索 API 时不做通用网页搜索。

## 8. 报告结构

每份报告固定包含：

### 8.1 诊断分

0-100 分，用于社交传播和快速比较。分数不是审美分，而是当前材料对外说服力、清晰度、信任度和行动路径的综合判断。

### 8.2 产品潜力分

0-100 分，判断这个产品是否值得继续投入。维度包括：

- 用户任务是否强。
- 问题是否高频、急、痛或值钱。
- 是否有外部需求信号。
- 替代方案是否明显。
- 差异化 wedge 是否成立。
- 分发路径是否现实。
- README/官网是否提供足够可信证据。

### 8.3 第一印象

一段具体判断，说明新用户第一次看到材料时会如何理解产品，以及哪里会卡住。

### 8.4 诊断标签

3-6 个标签，例如：

- 定位未收敛
- 用户任务不够清楚
- 价值主张弱于功能描述
- 缺少可信证据
- CTA 行动路径不明确

### 8.5 市场证据

3-6 条证据，区分：

- README / PDF 里明确写到的事实。
- 抓取网页得到的外部信息。
- 缺失的证据。
- Agent 基于材料做出的推断。

### 8.6 最大问题

3-5 个问题。每个问题包含：

- title：具体问题。
- why_it_matters：为什么影响理解、信任、激活、转化或产品感知。
- how_to_fix：具体怎么改。

### 8.7 推荐参考

推荐 3-5 个产品、品牌或体验参考。每个参考必须说明：

- 为什么与这个产品相关。
- 应该具体学习什么，而不是泛泛模仿风格。

### 8.8 下一步行动

5-10 条行动建议，按影响、确定性、成本和学习速度排序。

### 8.9 诊断边界

说明模型看不到或材料不足的地方，避免编造事实。

### 8.10 分享卡片

用于社交媒体发布，包含：

- 诊断分。
- 当前产品表达状态。
- 最大问题。
- 推荐参考。
- 一句可传播的诊断结论。

## 9. P0 功能

- 单入口聊天式首页。
- 单材料上传区。
- 支持 README/MD/TXT、PDF、PNG、JPG、WebP。
- 服务端 PDF 文本抽取。
- 服务端 README/TXT 文本抽取和 URL 提取。
- 抓取 README 中出现的公开网页。
- 可选 Serper 搜索。
- 图片本地指标抽取。
- DeepSeek 生成结构化产品诊断报告。
- 报告页展示材料预览、产品潜力、市场证据、诊断内容、参考对象、行动清单。
- 可观测 Agent 执行轨迹。
- 分享卡片 PNG 下载。
- 本地 JSON 存储分析记录。

## 10. P1 功能

- 网页 URL 抓取与截图。
- OCR 可见文本。
- 参考案例 web search。
- 报告质量 evaluator 自动修复。
- 邮箱收集和人工增强队列。
- 历史报告列表。
- 社交媒体分享落地页。
- 付费深度产品诊断。

## 11. 关键指标

流量指标：

- 社媒帖曝光。
- 页面访问。
- 评论、收藏、转发、私信数。

产品指标：

- 访问到上传转化率，目标 10%+。
- 上传到报告完成率，目标 80%+。
- PDF 上传占比。
- 报告页停留时长。
- 分享卡片下载率，目标 10%+。
- 人工诊断申请点击率。

定性信号：

- 用户是否说“这个说中了”。
- 用户是否主动上传真实产品材料。
- 用户是否根据报告修改产品介绍。
- 用户是否把报告发到社交媒体。
- 用户是否愿意付费让人继续诊断。

## 12. 发布文案

短版：

> 做了一个 Product Agent。上传产品介绍 PDF 或落地页截图，它会自动读材料，判断定位、用户任务、信任、转化和发布表达哪里有问题，然后给你下一步行动清单。

更尖锐版：

> 很多 AI 产品不是功能不够，而是用户看完不知道它到底帮谁、解决什么、为什么现在要用。我做了一个 Agent，上传产品介绍 PDF，它会帮你把这些问题拆出来。

征集版：

> 前 20 个产品我免费跑诊断。你只需要给我产品介绍 PDF 或落地页截图，我把报告发你。想测试一下它能不能说中真实问题。

## 13. 范围边界

第一版只解决：

> 用户上传产品材料后，知道产品表达和产品假设哪里最影响理解、信任、转化和下一轮信号。

第一版不做：

- 自动重设计完整页面。
- 长期课程。
- 多人协作。
- 复杂账号系统。
- 完整社区。
- 完全自治的开放任务 Agent。

## 14. 结论

Product Agent 的 MVP 不应该从“审美学习平台”开始，而应该从“产品材料诊断”开始。

这个入口更直接、更低成本，也更适合多产品同步发社交媒体测试反馈。用户只要上传材料就能得到结果，产品方也能通过上传率、分享率和付费意向快速判断是否有信号。
