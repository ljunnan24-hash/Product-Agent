# 开发任务拆解

## 当前产品原则

- 一个对话入口，一个上传材料处，后续由 Agent 自动读取、检索、判断和给下一步。
- 不展示隐藏思维链，展示可审计的工作流、工具调用、证据账本、证据边界和下一步实验。
- 减少 AI 主观判断，所有潜力判断都要绑定证据来源、证据时效、客观性和反证覆盖。

## P0：证据室闭环

状态：已完成。

- 新增 Evidence Brief 数据模型：EvidenceCard、ClaimLedger、EvidenceGap、EvidenceStop、ProductDecision。
- 新增本地证据引擎：把上传材料和网页调研拆成证据卡，生成关键假设账本。
- 接入上传分析 API：每次分析都会生成 evidenceBrief，并保存到本地分析记录。
- 接入 DeepSeek 报告提示词：报告必须参考 Evidence Brief，证据不足时不能高置信建议 build。

- 报告页新增“证据室”：展示当前决策、证据结论、证据停止原因、Claim Ledger 和下一步验证实验。
- 修复报告页时间格式 hydration mismatch。

## P1：调研能力增强

目标：把“查证据”从被动抓少量链接，升级为按假设主动调研。

状态：进行中，已完成第一段。

任务：

- Query Planner：根据目标用户、痛点、付费、替代方案、分发、反证分别生成查询。状态：已完成。
- Query Plan UI：在报告页 harness 中展示每条查询的意图、优先级、查询词和原因。状态：已完成。
- Multi-query Search：有 `SERPER_API_KEY` 时按优先级执行多条查询，结果绑定 `queryId`、`assumptionId` 和 `intent`。状态：已完成。
- Basic Recency Parser：从搜索结果日期、标题、snippet 中识别基础发布时间，并映射到 fresh / usable / historical。状态：已完成第一版。
- Opposition Candidate Search：主动生成失败、关闭、太贵、低频、不值得等反证查询。状态：已完成第一版。
- Source Budget：每个关键假设设置最低支持证据和反证要求，并显示当前缺口。状态：已完成第一版。
- Claim Evidence Detail：Claim Ledger 可展开查看支持证据、反证、相关预算和缺失证据。状态：已完成第一版。
- Evidence Normalizer：把搜索结果、爬取页面、用户材料统一归一成 EvidenceCard。状态：已完成第一版。
- Search Provider Adapter：支持 `zhipu` / `serper` 两种搜索 provider，默认优先使用智谱 Web Search。状态：已完成第一版。
- Report Model Provider：支持 `zhipu` / `deepseek` 两种报告模型，默认有智谱 key 时使用 `glm-5.2` 读材料和生成报告，DeepSeek 作为低成本/快速备选。状态：已完成第一版。
- Opposition Search v2：不只生成反证查询，还要抓取和归类失败原因、适用场景和是否仍然成立。状态：已完成第一版。
- Recency Parser v2：从抓取网页正文和 meta 标签识别发布时间/更新时间。状态：已完成第一版。
- Evidence Stop Rule v2：根据证据数量、来源多样性、时效性、反证覆盖和行为强度阻断强决策。状态：已完成第一版。
- Search Provider Quality Eval：对比智谱 / Serper 在 URL 完整性、反证命中率、中文产品语境和成本上的效果。状态：已完成第一版。
- Evidence-aware Research Loop：根据真实 Evidence Brief、Source Budget 和 Stop Rule 自动生成补查查询，执行后重算证据账本，并记录每轮状态、结果数、置信变化和停止原因。状态：已完成第一版。

- 历史报告列表第一版：新增 `/analyses` 报告库和 `GET /api/analyses` 摘要接口，可回看全部本地分析记录，展示潜力分、证据置信、质检状态、当前决策、材料数、网页证据数和首要修复问题；首页和报告页都增加报告库入口。
- Report Quality Evaluator 第一版：新增确定性报告质检规则，检查证据绑定、具体性、推断边界、实验闭环、时效性和 Source Budget 覆盖；分析完成和实验回填后都会生成 `reportQualityAudit`，报告页 harness 展示质量分、检查项、阻断问题和修复建议。
- Experiment Result 第一版：报告页可回填实验状态、样本量、主指标、证据摘要和原始链接；系统生成 `experiment_result` Evidence Card，更新 Evidence Brief、Source Budget、Claim Ledger、置信度和决策。
- Source Budget v2：根据首轮结果生成 budget-fill 查询；无搜索 key 时也显示补查计划并标记 skipped。
- Query Execution Trace：每条查询记录 planned / executed / skipped / failed、阶段、结果数和原因。
- Harness Query Status：调研计划展示首轮/补查、执行状态和结果数，避免把计划误读为证据。
- Zhipu Web Search Provider：接入智谱 Web Search，搜索结果进入 Evidence Brief，并对无 URL 摘要做可信度降权。
- Zhipu Report Model Provider：报告生成默认切到智谱 `glm-5.2`，并保留 `REPORT_MODEL_PROVIDER=deepseek` 的手动回退。
- Evidence UI Detail v2：Source Budget 可展开查看支持证据、反证、候选证据和对应查询。
- Opposition Search v2 第一版：反证归类为失败/关闭、无需求、低优先级、价格抗拒、免费替代、强竞品、迁移成本和分发风险。
- Recency Parser v2 第一版：抓取网页时解析 meta、JSON-LD、time 标签、正文日期和 Last-Modified；过滤明显未来日期，并把日期来源写入证据 caveat。
- Evidence Stop Rule v2 第一版：强决策阻断拆成外部证据、Source Budget、反证覆盖、证据时效、客观性、行为强度和来源多样性七条可审计规则，并在报告页展示规则状态和补证清单。
- Search Provider Quality Eval 第一版：每次搜索记录 query 成功率、URL 覆盖、日期覆盖、反证产出、假设覆盖、时效覆盖、摘要长度和 warning，并在报告页展示搜索质量。
- Experiment Schema 第一版：下一步验证实验增加 id、assumptionId、status、主指标、二级指标、证据采集清单、回填字段、通过/不确定/失败判定规则和 result 占位。
